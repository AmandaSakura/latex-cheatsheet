\section{L1 Foundations: Tokens, Unicode, BPE, Regex}
\subsection*{Token Basics}
Type: unique element in vocabulary $V$; size $|V|$.\\
Token (instance): one occurrence in corpus; total count $N$.\\
Lecture example sentence ``They ... stars'': 14 types, 16 instances (ignore punctuation).\\
Orthographic vs grammatical word can differ: ``I'm'' is one orthographic token\\
but two words ($I$ + $am$).\\
Heaps/Herdan law (lecture statement):
\[
|V|\approx KN^{\beta},\qquad \beta\approx 0.5.
\]
Type-token ratio $|V|/N$ decreases as corpus grows.

\subsection*{Language and Token Budget}
Whitespace tokenization is language-dependent; not universal.\\
For Chinese/Japanese/Thai, segmentation choices change token count strongly.\\
Chinese lecture point: words are built from hanzi; each character can correspond\\
to a morpheme.\\
Chinese segmentation is not uniquely agreed: same utterance can have multiple\\
valid word segmentations.\\
Practical exam note: same sentence meaning can consume very different\\
token budgets across tokenizers/languages.

\subsection*{L1 Morphology and ``What is a Word''}
Morpheme: minimal meaning-bearing unit in a language.\\
Examples from slides: ``fox'' = one morpheme; ``cats'' = $cat + {-s}$.\\
Root vs affix: root carries core meaning; affix adds grammatical/semantic function.\\
Inflectional morphemes: mostly grammatical\\
(e.g., verb past \texttt{-ed}, noun plural \texttt{-s/-es}).\\
Derivational morphemes: often change lexical class/meaning\\
(e.g., care $\rightarrow$ careful $\rightarrow$ carefully).\\
Clitic: syntactically word-like but phonologically reduced and attached\\
(e.g., \texttt{'ve} in \texttt{I've}).

\subsection*{Unicode, Code Points, UTF-8}
Unicode maps symbols to code points (e.g., \texttt{a}=U+0061).\\
Code point is not glyph; rendering belongs to font + shaping.\\
UTF-8 is variable length and ASCII-compatible:
\[
\begin{array}{l}
\text{1-byte: }0xxxxxxx,\\
\text{2-byte: }110xxxxx\ 10xxxxxx,\\
\text{3-byte: }1110xxxx\ 10xxxxxx\ 10xxxxxx,\\
\text{4-byte: }11110xxx\ 10xxxxxx\\
\text{\phantom{4-byte: }}10xxxxxx\ 10xxxxxx.
\end{array}
\]
ASCII (first 127 code points) uses 1 byte in UTF-8.\\
Python 3 strings are Unicode code points; \texttt{len()} counts code points (not bytes).

\subsection*{BPE (Byte Pair Encoding)}
Train-time learner (repeat for $k$ merges):
1. Find most frequent adjacent pair $(A,B)$.\\
2. Add merged token $AB$ to $V$.\\
3. Replace each $A\ B$ by $AB$.\\
Test-time encoder: start from chars/bytes.\\
Apply learned merges greedily in learned order.\\
Byte-level BPE uses alphabet size 256 (all bytes), avoiding byte-level OOV.

\subsection*{Regex Essentials (Tokenization/Cleanup)}
Character classes: \verb|[abc]|, \verb|[A-Z]|, \verb|[^0-9]|.\\
Quantifiers: \verb|*| (0+), \verb|+| (1+), \verb|?| (0/1), \verb|{m,n}|.\\
Anchors: \verb|^|, \verb|$|, boundary \verb|\b|, wildcard \verb|.|.\\
Grouping: capturing \verb|(...)|, non-capturing \verb|(?:...)|.\\
Lookahead: positive \verb|(?=...)|, negative \verb|(?!...)|.\\
Substitution API: \texttt{re.sub(...)}.\\
Capture-group reorder: group2 then group1.

\subsection*{L1 Pitfalls}
- Code switching: mixing languages in one utterance; tokenization counts can shift abruptly.\\
- Corpus datasheets: structured dataset docs for coverage, bias, intended use.\\
- Sentence boundary ambiguity: period can be abbreviation or EOS.\\
- Tokenizer mismatch invalidates direct perplexity/model comparison.\\
- Mixing byte-level and code-point operations causes silent bugs.
