\section{L1 Foundations: Tokens, Unicode, BPE, Regex}
\subsection*{Token Basics}
Type: element of vocabulary $V$; vocabulary size $|V|$.\\
Token/instance: one occurrence in running text; total instances $N$.\\
Corpus growth (Heaps/Herdan law): $|V| \approx K N^{\beta}$ with $\beta \approx 0.5$.\\
Practical note: token budget differs by language; non-English text is often split into more tokens.

\subsection*{Unicode and UTF-8}
Unicode assigns each character a code point (e.g., \texttt{a} is U+0061).\\
Code point range: U+0000 to U+10FFFF (about 1.1M possible).\\
UTF-8 is variable-length:\\
- ASCII range (first 127 code points): 1 byte.\\
- Other code points: 2, 3, or 4 bytes.\\
Python 3 strings are Unicode code points; \texttt{len()} counts code points, not bytes.

\subsection*{Byte Pair Encoding (BPE)}
\textbf{Learner (train-time):} repeat for $k$ merges:\\
1. Find most frequent adjacent pair $(A,B)$.\\
2. Add merged token $(AB)$ to vocabulary.\\
3. Replace all occurrences of $A\ B$ by $AB$ in corpus.\\
\textbf{Encoder (test-time):} start from character/byte sequence, then apply learned merges greedily in learned order.

\subsection*{Regex Essentials (for tokenization/pretokenization)}
Classes: \verb|[abc]|, ranges \verb|[A-Z]|, negation \verb|[^0-9]|.\\
Quantifiers: \verb|*| (0+), \verb|+| (1+), \verb|?| (0/1), \verb|{m,n}|.\\
Anchors: \verb|^| start, \verb|$| end, \verb|\b| word boundary, \verb|.| wildcard.\\
Groups: capturing \verb|(...)| for substitution \verb|re.sub(...)|; non-capturing \verb|(?:...)|.\\
Lookahead: positive \verb|(?=...)| and negative \verb|(?!...)| keep boundary conditions without consuming text.

\subsection*{L1 Pitfalls}
- Whitespace tokenization fails for many scripts (e.g., Chinese/Japanese/Thai).\\
- Periods are ambiguous (sentence end vs abbreviation/number).\\
- UTF-8 bytes and Unicode code points are different units; avoid mixing them in formulas/code.
